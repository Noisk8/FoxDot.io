Metodos del reproductor

Adem√°s de manipular a tus reproductores cambiando sus atributos, tambi√©n puedes utilizar m√©todos para cambiar sus comportamientos. Algunos m√©todos simplemente aplican un preset a atributos espec√≠ficos para ahorrar tiempo de escritura. Por ejemplo, el m√©todo spread equivale a ajustar la panor√°mica de un reproductor tanto a la izquierda como a la derecha y a compensar la afinaci√≥n, utilizando el atributo pshift, en 0,125 semitonos en cada canal:

# Difundiendo un sonido a trav√©s de canales est√©reo fuera de fase

d1 >> play("x-o-", pan=(-1, 1), pshift=(0, 0.125))

# Esto se puede hacer usando el m√©todo spread

d1 >> play("x-o-").spread()

# Las instrucciones se interpretan de izquierda a derecha, de modo que la dispersi√≥n anula el "pan=[-1, 0, 1]".

d1 >> play("x-o-", pan=[-1, 0, 1]).spread()

# Usando "spread" *antes* el >> establecer√° el paneo en[-1, 0, 1] pero pshift seguir√° siendo (0, 0.125)

d1.spread() >> play("x-o-", pan=[-1, 0, 1])

A continuaci√≥n se muestra una lista de m√©todos que se pueden utilizar con el c√≥digo de ejemplo adjunto. Esta lista est√° incompleta, as√≠ que por favor, ten paciencia conmigo mientras actualizo esta lista con m√°s informaci√≥n.

__add__(value)

Esto agrega valor al grado del Reproductor. Si est√° utilizando el play synthdef, el valor se a√±ade al atributo de muestreo en su lugar.

#Puede ser un patr√≥n, PGroup, var, o un solo n√∫mero

#El uso de 'play' alterar√° los valores de la muestra de manera que estas l√≠neas sean equivalentes

p2 >> play("x-o-", sample=[2, 3]) + [0, 0, 1]

M√©todos de instancia

stop()

Elimina el Reproductor del reloj de programaci√≥n para que ya no pueda escucharlo.

#Si un reproductor est√° sonando, use .stop() para detenerlo!

p1 >> pluck()

p1.stop()

#Tambi√©n puede a√±adir el m√©todo al final de la l√≠nea

reset()

Restablece todos los valores de atributo a sus valores originales. Esto es cero para todos los valores excepto oct, rate y amp, que son 5, 1 y 1 respectivamente.

#Inicia el reproductor

p1 >> play("x-o-", dur=1, rate=2, hpf=4000)

#Restablecer los atributos a los valores predeterminados

p1.reset()

#√ötil cuando no se desea ajustar manualmente los valores a 0, por ejemplo

p2 >> pluck([0, 2, 4, 7], dur=1/4, hpf=500, pan=[-1,1])

#Ajusta el dur a 1, hpf a 0 y pan a 0

p2.reset() >> pluck([0, 2, 4, 7])

solo(switch=1)

Silencia a todos los dem√°s Reproductores activos de manera que este Reproductor sea el √∫nico que se escuche. Para anular el silenciamiento de un reproductor individual, simplemente vuelva a evaluar una l√≠nea de c√≥digo. Para desactivar el silenciamiento de todos los reproductores, llame a este m√©todo con el conmutador puesto en 0 o Falso.

#Empieza con algunos reproductores

d1 >> play("x-o-")

p1 >> pluck([0,2], dur=PDur(3,8))

#Solo p1

p1.solo()

#Re-agregar otros reproductores

p1.solo(0)

only()

Detiene a todos los dem√°s reproductores activos. A diferencia de solo, este efecto no puede deshacerse usando s√≥lo(0) ya que la parada es permanente. Necesita reintroducir manualmente los reproductores.

#Empieza con algunos reproductores

d1 >> play("x-o-")

p1 >> pluck([0,2], dur=PDur(3,8))

p2 >> pads([0,3], dur=8, oct=4)

#Parar p1 y p2 con d1.only()

d1.only()

now(attr="degree")

Devuelve el valor actual (es decir, el √∫ltimo valor reproducido) para un atributo dado. Por defecto, √©ste es el grado (tono o car√°cter de ejemplo), pero puede especificar el nombre del atributo como una cadena.

#Comienza un reproductor

#Imprime el pitch y la duraci√≥n

print(p1.now(), p1.now("dur"))

print(p1.now(), p1.now("dur"))

4, 5

every(n, method_name, *args, **kwargs)

Util√≠celo para llamar a un m√©todo cada n n√∫mero de latidos. Especifique el nombre del m√©todo como una cadena y, a continuaci√≥n, los argumentos y los argumentos de palabra clave para proporcionar el m√©todo. Consulte Manipulaci√≥n algor√≠tmica para obtener informaci√≥n m√°s detallada sobre cada

#Comienza un reproductor

p1 >> pluck([0,1,2,3,4,5,6,7,8])

#Invierte el orden cada 8 latidos

p1.every(8, "reverse")

#Las llamadas a m√∫ltiples m√©todos pueden ser encadenadas entre s√≠

p1 >> pluck([0,2,4,7]).every(4, "reverse").every(8, "rotate")

after(n, method_name, *args, **kwargs)

Similar every, pero s√≥lo llama al m√©todo una vez despu√©s de n n√∫mero de latidos. Esto puede ser √∫til, por ejemplo, para reproducir en bucle una secuencia durante un tiempo determinado:

# deten un loop de drum despues de 8 latidos

d1 >> play("x-o-").after(8, "stop")

stutter(n, *args, **kwargs)

Repita el √∫ltimo evento musical n veces. Puede especificar argumentos de palabras clave, ya que normalmente actualizar√≠a un reproductor para controlar el tartamudeo. La palabra clave dur cambia la duraci√≥n durante la cual se extienden los eventos tartamudeados (por defecto es el valor de duraci√≥n actual del Jugador).

#comienza un reproductor

d1 >> play("x-o-")

d1.stutter()

#tartamudea 4 veces (3 eventos extra) d1.stutter(4)

#Tartamudear 8 veces m√°s de 2 latidos y paneo alternativo

d1.stutter(8, dur=2, pan=[-1,1])

#Muy √∫til cuando se combina con 'every'

d1.every(8, "stutter", 4, dur=3)

jump(ahead=1, **kwargs)

Reproduce el evento que est√° un n√∫mero de pasos por delante de la posici√≥n actual del reproductor, tal y como se especifica en el argumento ahead. Esto se utiliza mejor en conjunci√≥n con cada m√©todo:

#Toca el drum de la caja medio segundo despu√©s de 6.5 golpes

d1 >> play("x-o-").every(6.5, "jump", cycle=8)

spread(pshift=0.125)

Ajusta la panoramizaci√≥n tanto a la izquierda como a la derecha y compensa la afinaci√≥n en 0,125 semitonos de forma predeterminada para obtener un sonido est√©reo m√°s denso.

# sin spread

p1 >> pluck([0, 4], oct=4, dur=PDur(3,8), sus=2)

#con spread

p1 >> pluck([0, 4], oct=4, dur=PDur(3,8), sus=2).spread()

slider(start=0, switch=1)

Crea un efecto de diapositivas alternado con el reproductor. Fijar inicio a 1 para cambiar el orden de las diapositivas de arriba hacia abajo. Utilice switch=0 para desactivar el efecto o el m√©todo de reinicio.

#Reproductor basico

p1 >> saw([7, 0, 3, 1, 7, 4, 5, 2], dur=1/4, oct=4)

# con slider added

p1 >> saw([7, 0, 3, 1, 7, 4, 5, 2], dur=1/4, oct=4).slider()

penta(switch=1)

Establece la escala en el modo pentat√≥nico de la escala por defecto. Use switch=0 para volver a la escala por defecto.

#escala por defecto

p1 >> pluck([0, 1, 2, 3, 4, 5, 6, 7])

#pentatonica

p1 >> pluck([0, 1, 2, 3, 4, 5, 6, 7]).penta()

degrade(amount=0.5)

Establece la amplitud a elegir entre 1 y 0 de forma aleatoria. El argumento de la cantidad es la probabilidad de que la amplitud sea 0 como una fracci√≥n, es decir, una degradaci√≥n de 0,5 (por defecto) significa que cada evento tiene un 50% de probabilidad de ser un 0 y a 0,1 tiene un 10% de probabilidad de ser un 0.

#empieza un reproductor

p1 >> dirt ()

#Comenzar a disminuir el n√∫mero de eventos

p1.degrade()

#Puede ser llamado repetidamente para degradarse en un 25% cada 8 latidos

p1.every(8, "degrade", 0.25)

offbeat(n=1)

Establece el dur a n y el retardo a n / 2 para que cada nota se toque en el offbeat.

#Usa la percusi√≥n para escuchar la diferencia

d1 >> play("x-o-")

p1 >> pluck()

#Ajustar la duraci√≥n a 1 y offbeat

p1.offbeat()

#Ajuste la duraci√≥n a 2 y desact√≠vela cada 2 latidos p1.offbeat(2)

reverse()

Invierte el orden de todos los atributos en el momento actual. Este m√©todo no invierte las listas de valores, sino que invierte el orden en que se utilizan.

#toca una octava

p1 >> pluck([0, 1, 2, 3, 4, 5, 6, 7])

# Llamada inversa y las notas empiezan a descender

p1.reverse()

#Llamada usando "every" para movimiento org√°nico arriba y abajo

p1.every([6,3], "reverse")

rotate(n=1)

Mueve todos los elementos de la cartograf√≠a de afinaci√≥n/muestreo en n pasos de tal forma que la nueva lista de datos se inicia de forma equivalente, pero comienza en el en√©simo elemento de la lista original.

#empieza un reproductor

p1 >> pluck([0,1,2,3])

Rotando [0,1,2,3] cambiando esto por [1,2,3,0]

p1.rotate()

shuffle()

Aleatoriza el orden del atributo del car√°cter de afinaci√≥n / ejemplo.

follow(player)

Sigue el terreno de juego del reproductor. Es equivalente a ajustar la afinaci√≥n de un reproductor, por ejemplo, p1, a la afinaci√≥n de otro usando p2.pitch.

#comienza un reproductor

#siguiendo el pitch en p2

p2 >> blip().follow(p1)

#agregando valores de cambio al pitch

p2 >> blip().follow(p1) + [0, 2]

#Play acompa√±ando pitches

p2 >> blip().accompany(p1)

agregando valores para acompa√±ar el pitch
